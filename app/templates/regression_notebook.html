Regression Analysis to Find the Compressive Strength of Concrete¶

Professor I-Cheng Yeh data collected for for "Modeling of strength of high performance concrete using artificial neural networks," Cement and Concrete Research, Vol. 28, No. 12, pp. 1797-1808 (1998), is available as part of the University of California, Irvine, machine learning repository. The data is very suited to regression analysis.

Examining Data with a Biplot¶

Data analysis is a negotiation between mathematical accuracy and human understanding. A biplot is a good example of a bargain struck between those competing forces.

A biplot is a scatter plot of first and second principle component transformations of the data, onto which we project lines for each of the fields in the dataset, from the origin to the maximum value of the field. The x- and y-axes of our plot don’t really conform to the data fields themselves, but their correspondence is close enough for a human brain to get an idea of how the different fields interact with each other. R’s native plotting provides a biplot functionality by default – Bokeh will take a little more work, but produce, perhaps, a richer final chart.

Teddy Roland’s excellent biplot for matplotlib (https://github.com/teddyroland/python-biplot/blob/master/biplot.py) was a big help in making this chart.

Creating the Biplot¶

  1. A biplot consists of points and arrows. Therefore, we create two dictionaries, points and arrows to hold our data.
  2. The pca object has a property components_. We use this property to identify the first and second princple axes of our data, which we store in the variables pc1 and pc2.
  3. Calling .tranform() on our pca object with features_scaled as an argument returns a numpy array of reduced dimensionality. The data appropriate to our first and second principle axes are identified at indexes [:,0] and [:,0] of this reduced dataset.
  4. These four variables - pc1, pc2, first_transformation, and second_transformation give us all the data we need to create our arrow and scatter plots.
  5. Finally, we create our Bokeh figure p, and add the arrows, labels and scatter plot.

points = {'first_transformation': [],
         'second_transformation': []}

arrows = {'bottomX': [],
      'bottomY': [],
      'headX': [],
      'headY': [],
      'label': []}


pc1 = pca.components_[0] # first principle component
pc2 = pca.components_[1] # second principle component

first_transformation = pca.transform(features_scaled)[:,0] 
second_transformation = pca.transform(features_scaled)[:,1]

# prepare the data for plotting
for i in range(len(df.columns[:-1])):
    title = df.columns[i]
    arrows['bottomX'].append(0)
    arrows['bottomY'].append(0)
    arrows['headX'].append(pc1[i]*max(first_transformation))
    arrows['headY'].append(pc2[i]*max(second_transformation))
    arrows['label'].append(title)


[points['first_transformation'].append(i) for i in first_transformation]
[points['second_transformation'].append(i) for i in second_transformation];


p = bp.figure(height=600,
             width = 600,
             title="Biplot")

Points of interest from the Biplot¶

  • Water and fine aggregate are more-or-less inversely proportional to one another.
  • Fine and coarse aggregates are more-or-less orthogonal to each other.
  • Some of the fields are more-or-less co-linear, which we’ll note for later.

In meantime, we'll move on to the regression.

Regression¶

We’ll try five linear regression models from the sklearn suite of packages. Some of these models have variable parameters, some do not. We’ll use sklearn’s grid-search cross validator for models where we have variable parameters, and do without them where we don’t.

We’ll use kfold cross-validation to create the testing and training datasets. Finally, we’ll use the square root of the mean squared error as our metric of success – there are more detailed results available from GridSearchCV, but we’ll do without them for consistency, and because the data is relatively simple, as these datasets go.

The models are:

  1. Linear Regression
  2. Bayesian Ridge
  3. Lasso Regression
  4. Support Vector Regressor
  5. Decision Tree Regressor

    mse = mean_squared_error(predictions, labels)
    holder.append([a, np.sqrt(mse)])
